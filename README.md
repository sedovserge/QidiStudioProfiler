# QidiStudio Profiler

GUI-утилита для управления профилями филаментов в QIDIStudio.

Когда нужного типа филамента нет в студии (например, PETG-CF17 или PA6-CF20), приходится вручную создавать цепочку из трёх JSON-файлов, прописывать наследование, регистрировать в индексе и конфиге видимости. А для синхронизации с QIDI Box — ещё и вручную менять `filament_id` в JSON. Эта программа делает всё автоматически.

## Возможности

- **Создание** системных профилей филаментов (цепочка L1/L2/L3) на основе шаблона
- **Удаление** ранее созданных Advanced-профилей
- **Привязка filament_id** для синхронизации пользовательских прутков с QIDI Box

## Как это работает

### Проблема

QIDIStudio использует трёхуровневую систему профилей филаментов:

```
Level 1: базовый профиль серии        → Advanced PETG-CF25.json
Level 2: профиль принтера (Q2)        → Advanced PETG-CF25 @Q2.json
Level 3: профиль сопла (0.4 nozzle)   → Advanced PETG-CF25 @Qidi Q2 0.4 nozzle.json
```

Каждый файл должен иметь правильное наследование, уникальные ID, быть зарегистрирован в индексе (`Q Series.json`) и в конфиге видимости (`QIDIStudio.conf`). Ручное создание — рутинная и подверженная ошибкам работа.

### Что делает программа

1. Сканирует все базовые системные профили (~50 штук: Generic PETG, QIDI ABS, Bambu PLA и т.д.)
2. Пользователь выбирает базовый профиль как шаблон, вводит имя нового типа, отмечает нужные сопла
3. Программа создаёт все JSON-файлы с правильными параметрами:
   - Копирует настройки из шаблона (температуры, скорости, давление и т.д.)
   - Генерирует уникальные `filament_id` и `setting_id`
   - Настраивает наследование и совместимость
   - Устанавливает `filament_vendor: "Generic"` для корректного отображения в UI
4. Регистрирует новые профили в трёх местах:
   - `system/Q Series.json` → `filament_list` (индекс пресетов)
   - `QIDIStudio.conf` → `filaments[]` (реестр видимости)
   - Файлы на диске в `system/Q Series/filament/`
5. Делает бекап `QIDIStudio.conf` и `Q Series.json` в `backups/` перед каждой операцией

### Структура наследования

```
fdm_filament_common.json
├── Advanced PETG-CF25.json              (L1, instantiation: false)
│   name: "Advanced PETG-CF25@Q-Series"
│
├── Advanced PETG-CF25 @Q2.json          (L2, instantiation: false)
│   name: "Advanced PETG-CF25@Q2-Series"
│   inherits: fdm_filament_common        ← от common, не от L1!
│
└── Advanced PETG-CF25 @Qidi Q2 0.4 nozzle.json  (L3, instantiation: true)
    name: "Advanced PETG-CF25 @Qidi Q2 0.4 nozzle"
    inherits: "Advanced PETG-CF25@Q2-Series"      ← от L2
    compatible_printers: ["Q2 0.4 nozzle"]
```

> **Особенность QIDIStudio:** Level 1 и Level 2 оба наследуют от `fdm_filament_common` и содержат полный набор параметров. Только Level 3 наследует от Level 2 и содержит минимальные переопределения. Level 3 — единственный уровень, который виден пользователю (`instantiation: true`).

## Установка

### Из исходников

```bash
pip install PyQt5
python main.py
```

### Сборка exe

```bash
pip install pyinstaller
pyinstaller --onefile --windowed --name QidiStudioProfiler main.py
```

## Использование

### Создание профиля

1. **Закрыть QIDIStudio** (студия перезаписывает `QIDIStudio.conf` при выходе)
2. Запустить `QidiStudioProfiler.exe`
3. Выбрать базовый профиль — шаблон, из которого будут скопированы настройки печати
4. Ввести название нового типа (например, `PETG-CF25`)
5. Отметить нужные размеры сопел (по умолчанию 0.4 мм)
6. Нажать **«Создать профили»**
7. В диалоге подтверждения проверить список файлов → **«Подтвердить»**
8. Запустить QIDIStudio — новый тип появится в выпадающем меню филаментов под группой **Generic**

### Удаление профиля

1. **Закрыть QIDIStudio**
2. Запустить `QidiStudioProfiler.exe`
3. В секции «Удаление» выбрать профили для удаления (множественный выбор)
4. Нажать **«Удалить выбранные»** → подтвердить
5. Программа удалит файлы, записи из индекса и конфига видимости

### Привязка filament_id для QIDI Box

Системный профиль появится в студии, но для работы с QIDI Box нужен дополнительный **пользовательский профиль** с правильным `filament_id`.

1. В QIDIStudio: Настройки филамента → Создать пользовательский пруток → выбрать новый тип как основу
2. В секции **«QIDI Box: привязка filament_id»** выбрать пользовательский пруток
3. Выбрать производителя и тип филамента — программа автоматически подберёт значения по данным профиля
4. Проверить превью нового `filament_id` (формат `QD_1_<vendor>_<fila>`)
5. Нажать **«Применить»**

Программа считывает справочник из `officiall_filas_list.cfg` (vendor ID и filament type ID). Файл ищется в следующих местах:
1. Рядом с программой (рекомендуется)
2. Корень каталога QIDIStudio
3. `printers/` внутри QIDIStudio

Если файл не найден, секция QIDI Box будет отключена с подсказкой. Файл можно скопировать из прошивки принтера (`/usr/share/klipper/officiall_filas_list.cfg`).

## Бекапы

Перед каждой операцией (создание/удаление/привязка ID) программа сохраняет копии конфигов в `backups/`:

```
backups/
├── QIDIStudio_20260303_010500.conf
├── Q Series_20260303_010500.json
├── QIDIStudio_20260303_012030.conf
└── Q Series_20260303_012030.json
```

Для восстановления — скопировать нужный файл обратно, переименовав в оригинальное имя.

## Ограничения

- Создаёт профили только для **Q2** (Q1 Pro и Q2C не поддерживаются)
- Профили создаются как **системные** (`"from": "system"`) — будут потеряны при OTA-обновлении студии. Делайте бекапы
- В диалоге «Добавить/удалить пруток» студии Advanced-профили не отображаются (ограничение UI студии — фиксированный список вендоров). Но они доступны в выпадающем меню филаментов
