# QidiStudio Profiler

GUI-утилита для управления профилями филаментов в QIDIStudio.

Когда нужного типа филамента нет в студии (например, PETG-CF17 или PA6-CF20), приходится вручную создавать цепочку из трёх JSON-файлов, прописывать наследование, регистрировать в индексе и конфиге видимости. А для синхронизации с QIDI Box — ещё и вручную менять `filament_id` в JSON. Эта программа делает всё автоматически.

## Возможности

- **Создание** системных профилей филаментов (цепочка L1/L2/L3) на основе шаблона
- **Удаление** ранее созданных Advanced-профилей
- **Привязка filament_id** для синхронизации пользовательских прутков с QIDI Box

## Быстрый старт

### 1. Подготовка

Скачайте `QidiStudioProfiler.exe` из [Releases](../../releases) и положите его в **папку настроек QIDIStudio**:

```
C:\Users\<ваш пользователь>\AppData\Roaming\QIDIStudio\
```

> Чтобы быстро открыть эту папку: нажмите `Win+R`, введите `%appdata%\QIDIStudio` и нажмите Enter.

### 2. Скачайте справочник филаментов (для QIDI Box)

Если вы используете QIDI Box, скачайте файл `officiall_filas_list.cfg` с принтера и положите **рядом с программой** в ту же папку:

```bash
scp root@<IP принтера>:/usr/share/klipper/officiall_filas_list.cfg .
```

Или скачайте любым другим способом (WinSCP, SFTP-клиент и т.д.). Файл находится на принтере по пути `/usr/share/klipper/officiall_filas_list.cfg`.

Без этого файла секция привязки filament_id будет отключена. Для создания/удаления профилей он не нужен.

### 3. Запуск

1. **Закройте QIDIStudio** — студия перезаписывает конфиг при выходе, и ваши изменения пропадут
2. Запустите `QidiStudioProfiler.exe`
3. Программа автоматически найдёт каталог настроек студии и все профили

В итоге папка должна выглядеть примерно так:

```
C:\Users\<user>\AppData\Roaming\QIDIStudio\
├── QidiStudioProfiler.exe          ← программа
├── officiall_filas_list.cfg        ← справочник (опционально, для QIDI Box)
├── QIDIStudio.conf
├── system/
│   └── Q Series/
│       └── filament/
└── user/
    └── default/
        └── filament/
```

## Использование

### Создание профиля

1. Выбрать базовый профиль — шаблон, из которого будут скопированы настройки печати
2. Ввести название нового типа (например, `PETG-CF25`)
3. Отметить нужные размеры сопел (по умолчанию 0.4 мм)
4. Нажать **«Создать профили»**
5. В диалоге подтверждения проверить список файлов → **«Подтвердить»**
6. Запустить QIDIStudio — новый тип появится в выпадающем меню филаментов под группой **Generic**

### Удаление профиля

1. В секции «Удаление» выбрать профили для удаления (множественный выбор)
2. Нажать **«Удалить выбранные»** → подтвердить
3. Программа удалит файлы, записи из индекса и конфига видимости

### Привязка filament_id для QIDI Box

Системный профиль появится в студии, но для работы с QIDI Box нужен дополнительный **пользовательский профиль** с правильным `filament_id`.

1. В QIDIStudio: Настройки филамента → Создать пользовательский пруток → выбрать новый тип как основу
2. Закрыть студию, запустить QidiStudio Profiler
3. В секции **«QIDI Box: привязка filament_id»** выбрать пользовательский пруток
4. Выбрать производителя и тип филамента — программа автоматически подберёт значения по данным профиля
5. Проверить превью нового `filament_id` (формат `QD_1_<vendor>_<fila>`)
6. Нажать **«Применить»**

## Как это работает

### Трёхуровневая система профилей

QIDIStudio использует цепочку из трёх JSON-файлов для каждого типа филамента:

```
Level 1: базовый профиль серии        → Advanced PETG-CF25.json
Level 2: профиль принтера (Q2)        → Advanced PETG-CF25 @Q2.json
Level 3: профиль сопла (0.4 nozzle)   → Advanced PETG-CF25 @Qidi Q2 0.4 nozzle.json
```

Каждый файл должен иметь правильное наследование, уникальные ID, быть зарегистрирован в индексе (`Q Series.json`) и в конфиге видимости (`QIDIStudio.conf`). Программа делает всё это автоматически.

### Структура наследования

```
fdm_filament_common.json
├── Advanced PETG-CF25.json              (L1, instantiation: false)
│   name: "Advanced PETG-CF25@Q-Series"
│
├── Advanced PETG-CF25 @Q2.json          (L2, instantiation: false)
│   name: "Advanced PETG-CF25@Q2-Series"
│   inherits: fdm_filament_common        ← от common, не от L1!
│
└── Advanced PETG-CF25 @Qidi Q2 0.4 nozzle.json  (L3, instantiation: true)
    name: "Advanced PETG-CF25 @Qidi Q2 0.4 nozzle"
    inherits: "Advanced PETG-CF25@Q2-Series"      ← от L2
    compatible_printers: ["Q2 0.4 nozzle"]
```

> **Особенность QIDIStudio:** Level 1 и Level 2 оба наследуют от `fdm_filament_common` и содержат полный набор параметров. Только Level 3 наследует от Level 2 и содержит минимальные переопределения. Level 3 — единственный уровень, который виден пользователю (`instantiation: true`).

## Бекапы

Перед каждой операцией программа сохраняет копии `QIDIStudio.conf` и `Q Series.json` в `backups/` с меткой времени. Для восстановления — скопировать нужный файл обратно, переименовав в оригинальное имя.

## Сборка из исходников

```bash
pip install PyQt5
python main.py
```

Сборка exe:

```bash
pip install pyinstaller
pyinstaller --onefile --windowed --name QidiStudioProfiler main.py
```

## Ограничения

- Создаёт профили только для **Q2** (Q1 Pro и Q2C не поддерживаются)
- Профили создаются как **системные** (`"from": "system"`) — будут потеряны при OTA-обновлении студии. Делайте бекапы
- В диалоге «Добавить/удалить пруток» студии Advanced-профили не отображаются (ограничение UI студии — фиксированный список вендоров). Но они доступны в выпадающем меню филаментов
